#### Tab 4：归因分析

## 五、预测模型推荐与业务集成建议

### 5.1 业务预测场景梳理

| 预测场景 | 频率 | 数据特征 |
|----------|------|----------|
| 月度 KPI 目标（销量/利润） | 月度 | 有明显季节性和趋势 |
| 产品/渠道销量趋势 | 周/月 | 多维度、可叠加促销因素 |
| 资金/库存消耗预测 | 实时/日 | 受入库出库节奏影响 |
| 渗透率/留存率趋势 | 月度 | 相对平稳，波动小 |

### 5.2 推荐预测模型

#### 经典统计模型（优先推荐，门槛低）

| 模型 | 适用场景 | 工具 |
|------|----------|------|
| **Prophet**（Facebook 开源） | 月/季度 KPI 趋势、含节假日效应 | Python `prophet` 库 |
| **ARIMA / SARIMA** | 有季节性的时间序列（月销售额、渗透率） | Python `statsmodels` |
| **Holt-Winters 指数平滑** | 常规销售/业绩数据，快速简单 | Python `statsmodels` |

#### 机器学习模型（多变量场景）

| 模型 | 适用场景 | 工具 |
|------|----------|------|
| **XGBoost / LightGBM** | 含促销、流量、库存等多维特征的精度提升 | Python `xgboost` / `lightgbm` |
| **随机森林回归** | 数据维度适中、关系较明朗的指标预测 | Python `sklearn` |
| **LSTM / RNN** | 大量异动、复杂时序（开发维护成本较高） | Python `tensorflow` / `pytorch` |

#### AI 云平台（零代码，开箱即用）
- 阿里云 PAI 时序预测
- Amazon Forecast
- Google AutoML Tables

### 5.3 分场景推荐策略

```
简单 KPI/月度趋势
  └── 首选 Prophet 或 Holt-Winters，快速上线

需要解释多维影响因子（活动、渠道、促销）
  └── XGBoost / 多元回归 + SHAP 归因解释

复杂多变量、跨品类、异常波动多
  └── LSTM + 特征工程（需专业数据科学支持）

节假日/促销影响显著
  └── Prophet（内置节假日配置）
```

### 5.4 前端展示集成建议

- **KPI 卡片 / 趋势图下方**：显示"下月预测值 / 区间预测（置信区间）"，异常时高亮
- **各 Tab 增加"智能预测"区块**：展示未来 1~3 个月趋势线，与历史曲线叠加对比
- **支持一键跳转"预测精细拆解"视图**：按产品/渠道/区域分维度预测拆分
- **预测结果与异常告警联动**：若预测值偏离目标超过阈值，自动触发告警

### 5.5 Python 快速启动参考

```python
# 使用 Prophet 进行月度 KPI 预测（快速示例）
from prophet import Prophet
import pandas as pd

# 准备数据：ds（日期列）+ y（指标值列）
df = pd.DataFrame({
    'ds': pd.date_range('2024-01', periods=24, freq='MS'),
    'y': [100, 110, 105, ...]  # 历史月度销售额
})

model = Prophet(
    yearly_seasonality=True,
    weekly_seasonality=False,
    daily_seasonality=False
)
model.fit(df)

# 生成未来 3 个月预测
future = model.make_future_dataframe(periods=3, freq='MS')
forecast = model.predict(future)

# 提取预测结果
print(forecast[['ds', 'yhat', 'yhat_lower', 'yhat_upper']].tail(3))
```
