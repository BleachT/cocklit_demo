# 业务共创地图模板及行业看板获取指南

> 适用于 cockpit_demo 项目团队日常业务需求梳理与数据能力沉淀，帮助团队主动发现场景价值、沉淀可复用能力模块。

---

## 一、业务共创地图模板

### 使用说明

- **填写频率**：建议每次业务共创会议后更新，或每月例行梳理一次。
- **责任人**：每行分配明确的数据/业务负责人，避免多人无人认领。
- **优先级参考**：高（P0 当月推进）、中（P1 季度内）、低（P2 日后排期）。
- **进展备注**：记录最新推进状态、阻碍因素、下一步行动。

## 二、如何高效获取行业看板参考

在输出原型和方案之前，参考行业最佳实践的行业看板可以极大提升效率和说服力。以下分三个维度整理获取方式。

---

### A. 通过主流 BI 工具内置模板与社区获取

| 工具 | 获取方式 | 推荐路径 |
|------|---------|---------|
| **Power BI** | 应用市场（AppSource）内置行业模板 | [appsource.microsoft.com](https://appsource.microsoft.com) → 搜索"Power BI template"，按行业分类浏览 |
| **Tableau** | Tableau Public 社区图库 | [public.tableau.com/gallery](https://public.tableau.com/gallery) → 按主题/行业过滤 |
| **Domo** | Domo AppStore 内置行业解决方案 | Domo 后台 → AppStore → Industry Solutions |
| **阿里云 Quick BI** | 数据大屏模板市场 + 阿里云官方行业方案 | Quick BI 控制台 → 模板中心 / 阿里云官网 → 行业解决方案 |
| **帆软 FineReport/FineBI** | 帆软论坛模板分享区 + 官方行业案例库 | [bbs.fanruan.com](https://bbs.fanruan.com) → 模板下载 / 官网 → 行业案例 |
| **腾讯云 BI（腾讯云图）** | 腾讯云官方行业模板 + 腾讯文档图表 | 腾讯云控制台 → 云图 → 行业模板 |
| **Metabase** | 官方 Demo 和社区看板分享 | [metabase.com/community](https://www.metabase.com) |
| **Grafana** | Grafana Dashboard 社区（侧重技术/运维场景） | [grafana.com/grafana/dashboards](https://grafana.com/grafana/dashboards) |

**使用建议**：
- 直接导入模板后修改数据源，快速出原型效果。
- 重点关注与自己业务领域相近的行业标签（零售、金融、制造、互联网等）。
- 社区用户分享的真实看板往往比官方模板更贴近实际业务场景。

### C. 搜索关键词建议与优劣判断要点

#### 推荐搜索关键词

**中文关键词：**
```
行业名称 + 数据看板 / 数据驾驶舱 / KPI 指标体系
零售 / 电商 / 制造 / 金融 + 数字化运营看板
数据可视化 + 行业案例 + 模板
驾驶舱设计 + 指标体系 + 业务需求
```

**英文关键词：**
```
[industry] KPI dashboard template

sales / supply chain / retail / finance dashboard design
```

**平台组合推荐：**
- Google + "site:tableau.com" / "site:powerbi.microsoft.com"
- 国内推荐：百度 + 知乎专栏、CSDN、36kr 数据频道

---

#### 行业看板优劣判断要点

| 判断维度 | 优质看板特征 | 劣质看板警示 |
|---------|------------|------------|
| **指标相关性** | 指标直接关联业务核心目标（销售、利润、效率等） | 指标堆砌，缺乏业务逻辑主线 |
| **层次清晰度** | 从总览→趋势→明细，有清晰的下钻逻辑 | 所有信息平铺，无主次之分 |
| **可操作性** | 看板能指向具体业务动作（告警触发、流程审批等） | 仅展示数据，无业务决策指引 |
| **数据新鲜度** | 明确标注数据更新频率（实时/T+1/周报等） | 时效不明，可能误导决策 |
| **受众适配性** | 针对管理层/一线运营/分析师分层设计 | 通用化，不区分使用角色 |
| **交互体验** | 支持筛选、下钻、导出，操作符合直觉 | 静态图表，无法交互探索 |
| **行业背景契合** | 包含行业专属指标（如零售 GMV、制造 OEE、金融 NPS） | 通用指标，缺乏行业深度 |

## 三、综合使用建议

1. **建立共创地图 → 沉淀场景库**：每次业务共创后更新共创地图表格，定期（每月）复盘，将碎片需求归类到业务主题，防止需求碎片化。

2. **看板参考 → 启发原型设计**：在动手画原型前，先用上述渠道找 2~3 个相近行业的优质看板参考，提炼适合自身业务的指标结构和页面布局。

3. **原型驱动共创 → 快速迭代**：用 mock 数据快速搭建原型展示效果，在业务例会/场景评审中演示，结合共创地图记录反馈，形成"发现痛点→场景原型→业务共创→能力沉淀"的正向循环。

4. **能力产品化**：原型验证成功后，把可复用的指标口径、数据逻辑、可视化组件沉淀为标准件，支撑后续更快速的场景复制与扩展。
