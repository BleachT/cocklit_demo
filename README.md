## 📊 驾驶舱原型功能

当前原型包含：

- **核心 KPI 看板** — 销售额、毛利、T3 完成率、提车量实时监控
- **量价拆解** — 自动分解目标差距中"量"与"价"的贡献占比
- **渠道分布** — 终端 / APP / 其他渠道销售构成
- **产品线下钻** — 支持按分产品（车生长/车改装/车生活）下钻查看

## 🗺️ 业务背景 & 下一步建议

### 本轮业务沟通关键结论

1. **以选装为例的分析路径**：销售额下降 → 先看产品线占比 → 再下钻版型/颜色维度
2. **产品周期较长（5-6个月）**：调整决策需多方开会讨论，数据分析对"即时行动"指导意义有限
3. **业务核心诉求**：驾驶舱首要功能是**目标进度监控**，而非深度归因分析

### ✅ 针对上述情况的建议

| 情况 | 建议 |
|------|------|
| "目标监控就够了，分析太深没用" | 把驾驶舱 Level 1 做成**目标 vs 实际进度条 + 预警灯**，简洁有力 |
| 产品调整周期 5-6 个月 | 在驾驶舱中加入**趋势预测**模块，提前 1-2 个月给出预警，让决策层有足够提前量 |
| 下钻到颜色/版型层级 | 保留下钻能力，但**不放在首页**，作为"深入分析"入口折叠 |
| 分析结果需开会讨论 | 在驾驶舱中加**导出报告**功能，方便直接带到会议中讨论 |

### 📋 推荐的下一步行动

- [ ] **第一优先级**：完善 Level 1 目标完成率看板（T3 完成率 + 各产品线进度）
- [ ] **第二优先级**：加入预警机制（当某产品线进度落后超过 X% 时，自动标红）
- [ ] **第三优先级**：与业务对齐数据源，获取样例数据
- [ ] **第四优先级**：搭建含真实数据的交互原型，准备下次业务评审
